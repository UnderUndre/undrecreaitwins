## Summary

The **spec is genuinely strong** ŌĆö 13 FRs, tiered remediation, explicit cost/latency/fidelity NFRs, a thorough edge-case list, and honest clarifications. The problem is everything downstream: **`plan.md` is an unfilled template** (Constitution Check rubber-stamped, no Phase-0/Phase-1 design, `research.md`/`data-model.md`/`quickstart.md` are title-only stubs, `contracts/` missing), its file paths are **fictional** (don't match the real codebase), and `tasks.md` collapses the whole feature into 4 tasks that omit three FRs. On top of that, a core requirement the spec itself deferred (FR-013, agentic path) was **never decided** ŌĆö and `spec.md` is still marked `pre-clarify`. The headline safety property ("preserve facts/prices/names") is also **overstated**: the proposed structural fidelity check can't enforce it. Implement must not proceed.

## Alternative approaches considered

- **Placeholder masking instead of detect-then-regenerate for fidelity (F4).** Mask numbers/prices/dates/URLs/code as tokens (`__NUM0__`, `__PRICE1__`) before the translate call, restore verbatim after. This *guarantees* preservation of the critical tokens rather than detecting corruption post-hoc and paying for a regenerate. Cheaper and stronger for the price-safety requirement than the proposed structural compare + escalation. The existing validator already masks code/URL/email (`language-guard.ts:49-52`) ŌĆö the machinery is half there.
- **Local langid library for inbound (F6/cost).** The clarify chose "cheap LLM" for accuracy, but inbound mirror detection on *every* message is the cost driver the spec frets about (NFR-1, edge "cost-ą▒ąŠą╝ą▒ą░"). A local library (fastText/CLD3/franc) + confidence threshold handles the common case at $0/0-latency, escalating to the LLM only on low confidence ŌĆö which also directly serves the short-message edge case (`spec.md:89`). Worth weighing as a hybrid, not dismissing wholesale.
- **Structured-output langid** (`{lang, confidence}` via constrained decoding) ŌĆö gives a reliable parse (closes part of F9) and a confidence signal to drive the sticky/low-confidence logic (F12), instead of free-text the code must regex.
- **Edit-in-place streaming on capable channels.** Buffered delivery (FR-008) kills streaming for *all* guard-active personas, even happy-path. On channels that support message edit (Telegram/Slack), stream optimistically then edit on violation ŌĆö partial relief for the UX hit without abandoning correctness. Not free, but a middle path the spec ruled out as "physically impossible" (true only for non-editable channels).

# Спецификация 006: MTProto Channel (Telegram Userbot)

## 1. Описание

Адаптер для подключения Telegram-юзерботов к ядру (Engine). Позволяет цифровым двойникам общаться от лица реального пользователя через MTProto, а не через Bot API.

## 2. Архитектура

- **Библиотека:** GramJS или MTKruto.
- **Интеграция с Engine:** Реализует интерфейс `IChannelAdapter`.
- **Управление сессиями:** Сессии (session strings) передаются из Product (`ai-twins`) слоя, Engine лишь поднимает клиент и обрабатывает события `newMessage`.

## 3. Требования к реализации

- Обработка входящих сообщений (фильтрация только нужных чатов/пользователей).
- Контроль rate limits (FloodWait), чтобы не получить бан аккаунта. При получении FloodWait сообщения помещаются в очередь (лимит 50). Если ожидание больше 60 секунд — новые сообщения для этого чата дропаются.
- Индикация "Печатает..." во время генерации ответа.

## 4. Связь с Product

Product-слой (`ai-twins/specs/mtproto-session`) отвечает за логин (телефон, код, 2FA) и сохраняет строку сессии, которая затем передается сюда при инициализации `TwinChannel`.

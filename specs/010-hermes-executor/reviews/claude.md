## Summary

This is a well-structured and unusually thorough spec for an agentic LLM backend. The "engine as nervous system + safety interlock, Hermes as brain" boundary is crisply defined, and the always-agent (C2) + real write-actions (C1) decisions are honestly assessed for risk. The spec's headline strength is the **engine-mediated tool-gateway** pattern — the agent never holds secrets, never reaches the world directly. The headline weakness is that **Honcho is treated as a given dependency despite being unproven for multi-tenant production use**, and several concurrency/ordering invariants around write-action idempotency are stated but not mechanically enforced in the data model or contract.

## Alternative approaches considered

1. **Local tool execution vs. engine-mediated gateway**: The spec commits to engine-mediated (agent calls back through engine). An alternative is allowing the agent to execute read-tools directly (lower latency, simpler) while mediating only write-tools. This wasn't explored and may significantly reduce round-trip latency for read-heavy agent loops. Worth weighing — the current design adds at least 2 RTTs per tool call (agent→engine→tool→engine→agent).

2. **Honcho vs. engine-native working memory**: Honcho is adopted as "Hermes-native" but this creates a new external dependency. An alternative is storing working memory in engine Postgres (JSONB column on `conversations` or a dedicated table) — simpler, fewer moving parts, no reconstruction needed. The trade-off is less sophisticated memory operations. The spec doesn't explain why this was dismissed beyond "adopt Honcho."

3. **Agent loop in-engine (TypeScript) vs. delegating to Hermes (Python)**: The current design delegates the entire agentic loop to Hermes. An alternative is implementing the loop in-engine (TypeScript) using OmniRoute for LLM calls, which would eliminate the TS↔Python integration surface, remove Hermes as a dependency, and simplify lifecycle. The trade-off is reimplementing tool-calling loop logic. The spec doesn't explicitly reject this — it assumes Hermes as given.

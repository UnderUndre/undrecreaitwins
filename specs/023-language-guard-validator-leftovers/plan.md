# Implementation Plan: Language Guard Validator Leftovers — API, Config Fields, Audit Log

**Branch**: `specs/023-language-guard-validator-leftovers` | **Date**: 2026-06-19 | **Spec**: `specs/023-language-guard-validator-leftovers/spec.md`
**Input**: Feature specification from `/specs/023-language-guard-validator-leftovers/spec.md`

## Summary

`LanguageGuardValidator` (259 LOC) is fully implemented and registered in the pipeline, but **exposes no HTTP API** for configuration management, lacks `enabled` toggle and `configVersion` optimistic locking, and has no audit log read endpoint. This plan adds 4 components: (1) `GET/PUT /v1/personas/:id/validators/language-guard` for config CRUD, (2) `enabled: boolean` field with pipeline skip (also gates system-prompt directive injection in `chat-service.ts`), (3) `version INTEGER` DB column for atomic optimistic locking (single-statement UPSERT, no TOCTOU), (4) `GET .../logs` audit log endpoint with compound cursor pagination. Tenant resolution reuses the existing global `onRequest` hook in `server.ts`.

## Technical Context

**Language/Version**: TypeScript 5.7+, strict mode, ESM only
**Primary Dependencies**: Fastify (HTTP framework), Zod (validation), Drizzle SQL templates (raw SQL via `sql` tagged template — not Drizzle query builder)
**Storage**: PostgreSQL (via `validator_configs` table for config, `validator_runs` for audit logs)
**Testing**: Vitest unit + integration
**Target Platform**: Node.js ≥20 server
**Project Type**: Web service (API layer for existing engine)
**Performance Goals**: GET config < 50ms, PUT config < 100ms, audit log query < 200ms (measurement: manual verification via `curl` timing or `autocannon` bench — no automated perf gate in initial implementation)
**Constraints**: Tenant-scoped via existing global `onRequest` hook in `server.ts` (resolves tenant from `x-tenant-claim` with `x-tenant-id` fallback). One DB migration: new `version INTEGER` column on `validator_configs`. Reuse existing `validator_configs` and `validator_runs` tables. Backward-compatible with existing configs missing `enabled`.
**Scale/Scope**: Per-(tenant, persona) validator config. Audit log paginated, indexed on `(validatorName, createdAt DESC, id DESC)`.
